# タスク管理議論

## 背景

日々の実施した内容のレポートをマークダウン形式で作成し、記憶装置として使いたいという提案から始まった。

## 検討した選択肢

### 1. シンプルな日報（1日1ファイル）

```markdown
# YYYY-MM-DD

## 完了
- [x] タスクA

## 進行中
- [ ] タスクB

## 気づき
- 〜について〜だと分かった
```

### 2. 日付フォルダ + トピック別ファイル

```
docs/2026/02/24/
├── index.md        ← 概要
├── topic-a.md      ← 詳細
└── topic-b.md
```

## 結論

**ハイブリッド採用:**
- 日報は「目次」として機能
- 詳細な技術メモは別ファイルに分割
- VitePressで人間が見やすく、MarkdownでAIが読み書きしやすい

## 利点

- テキストベースで検索しやすい
- Gitで履歴が残る
- AIが読み書きしやすい
- シンプルで道具に依存しない
